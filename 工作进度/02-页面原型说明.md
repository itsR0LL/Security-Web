# Security Studio 页面原型说明 v2.0

更新时间：2026-06-03

## 1. 页面原则

当前页面风格继续保持森雨式个人化表达：斜向结构、准星、细线、扫描、暗色背景、白灰主色、浅绿点缀。

功能页必须保持可用性：

- 首页不堆运营面板。
- 仿真页不放无关长列表。
- 事件页承担筛选与证据查看。
- 设置页承担数据源配置。
- 分析页承担攻击聚合与规则建议草案。

## 2. `/security`

首页定位为入口与主视觉。

保留内容：

- 粒子地球。
- 成都目标点。
- 斜向个性导航。
- 飞线 hover 小窗口。
- 少量状态信息。

本期未修改首页布局、地球、飞线和开屏动画。

## 3. `/security/situation`

态势页定位为安全态势仿真与侧栏摘要。

保留内容：

- 3D / 2D 切换。
- 粒子地球和 2D 地图。
- 拖拽、缩放、飞线 hover。

二期新增侧栏内容：

- 分析摘要。
- 攻击类型趋势。
- 规则命中趋势。
- 来源习惯。
- 数据可信度。

侧栏中的攻击类型、来源、规则可跳转到事件页筛选。

## 4. `/security/events`

事件页定位为证据工作台。

当前结构：

- 左侧筛选。
- 中间事件列表。
- 右侧事件详情。

当前筛选支持：

- 时间范围。
- 风险等级。
- 事件类型。
- 攻击分类。
- 规则 ID。
- 来源 IP。
- 国家/地区。
- 路径。
- User-Agent。
- 方法。
- 状态码。
- Cloudflare action。

二期已修正：从分析页跳转携带的 `attackCategory`、`ruleId`、`method`、`userAgent` 会进入服务端请求与页面初始筛选。

## 5. `/security/analysis`

分析页是二期新增核心页面。

页面结构：

- 顶部标题与全局导航。
- 左侧攻击行为组列表。
- 中间攻击类型矩阵、证据摘要、规则命中趋势。
- 右侧分析摘要、来源习惯、数据可信度、规则建议草案。

页面数据来自：

- `GET /api/analysis/summary`
- `GET /api/analysis/clusters`
- `GET /api/analysis/rules`
- `GET /api/analysis/sources`
- `GET /api/analysis/advice`

规则建议只展示草案，不写入、不启用。

## 6. `/security/settings`

设置页继续承担：

- Cloudflare Zone ID。
- Cloudflare API Token。
- 监控域名。
- 同步周期。
- 高风险阈值。
- 原始事件保留天数。
- Token 权限状态。

Token 不回显。

## 7. 当前页面验收状态

已验证：

- `/security/analysis` 返回 200。
- `/security/events?attackCategory=edge_security&ruleId=builtin-cloudflare-action` 返回 200。
- `/security/situation` 返回 200。
- `npm run build` 路由表包含 `/security/analysis`。

受限项：

- 内置 Playwright 浏览器 profile 被占用，本轮未完成真实浏览器截图验收。
- 已使用 HTTP 返回、构建、TypeScript、lint 和 API 冒烟替代检查。
