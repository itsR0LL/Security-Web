# Security Studio 页面原型说明 v1.0

更新时间：2026-06-02

## 1. 页面设计原则

Security Studio 的页面风格继续靠近森雨网站的个性化表达，但不能因为个性化影响功能使用。

页面规则：

- 首页负责个性化入口和主视觉。
- 安全态势页负责仿真与态势控制。
- 事件页负责证据、筛选、详情、告警文本。
- 设置页负责数据源与同步配置。
- 后续分析页负责攻击类型、规则命中、攻击习惯、异常聚合。

不再把所有功能塞到首页，也不再在每个页面重复放仿真框。

## 2. 当前页面结构

### `/security`

定位：个性化首页与安全入口。

当前保留：

- 粒子地球主视觉。
- 成都目标点。
- 森雨风格斜向导航。
- 准星、扫描线、角标、斜栏 hover。
- 少量主视觉状态信息。
- 飞线 hover 小窗口。

首页不再放：

- 大量 KPI 卡片。
- 事件长列表。
- 运营面板。
- 复杂筛选器。
- 详细分析表格。

### `/security/situation`

定位：安全态势仿真页。

当前保留：

- 3D / 2D 态势切换。
- 地图粒子可视化。
- 飞线与来源点。
- 飞线 hover 小窗口。
- 态势控制区域。

下一步增强：

- 左侧增加筛选与图层控制。
- 右侧增加攻击态势分析摘要。
- 点击来源、飞线、分类后可跳转事件页或分析页。

### `/security/events`

定位：事件证据页。

页面结构应保持三栏：

- 左侧：筛选。
- 中间：事件列表。
- 右侧：事件详情与待发送告警文本。

筛选项必须覆盖：

- 时间范围。
- 风险等级。
- 事件类型。
- 来源 IP。
- 国家/地区。
- 请求路径。
- User-Agent。
- 方法。
- 状态码。
- Cloudflare action。

详情区必须覆盖：

- 基础请求信息。
- 来源定位。
- Cloudflare 处理结果。
- 命中规则。
- 原始 JSON。
- AI 分析占位。
- 待发送告警文本。

### `/security/events/[id]`

定位：单事件详情页。

要求：

- 使用 `GET /api/events/{event_id}` 读取详情。
- 不通过拉全量列表再本地筛选。
- 展示证据链、原始字段、规则命中、告警文本。

### `/security/settings`

定位：数据源与安全策略设置页。

页面结构：

- 左侧：当前数据状态摘要。
- 中间：配置表单。
- 右侧：权限、样例模式、数据边界说明。

表单覆盖：

- Cloudflare Zone ID。
- Cloudflare API Token。
- 监控域名。
- 同步周期。
- 高风险阈值。
- 原始事件保留天数。

Token 不回显，只显示是否已配置。

### `/security/map`

定位：请求分布地图页。

当前作为 `/security/situation` 的补充页面使用，主要展示请求分布与 2D 地理态势。

## 3. 下一步新增页面

### `/security/analysis`

定位：攻击行为分析页。

这是下一阶段最重要的新页面。

建议模块：

- 攻击类型矩阵。
- 来源地区攻击习惯。
- 工具指纹摘要。
- 规则命中趋势。
- 未知攻击聚合。
- AI 规则建议。
- 规则版本记录。

页面不做成企业仪表盘，要继续保持当前个性化风格。内容区可以采用窄栏、分区、扫描线、斜向标题，但数据必须清楚。

## 4. 首页仿真框信息分配

### 内侧小框

内侧小框只放与当前地球仿真强相关的信息。

建议展示：

- `CHENGDU TARGET`
- `r0l1dehome.asia`
- 当前数据模式：`LIVE` / `SAMPLE` / `FALLBACK` / `DEGRADED`
- 当前时间窗口：`24H` / `7D` / `LAST SYNC`
- 当前筛选：`ALL` / `VISIT` / `ATTACK` / `ANOMALY`

不放长文本、不放表格、不放事件列表。

### 外侧主视角框

外侧主视角框放整体状态摘要。

建议展示：

- 请求量。
- 数据吞吐量。
- 高风险事件数。
- Cloudflare 拦截 / 质询数量。
- 来源国家、地区、城市数量。
- 缓存命中率。
- 当前同步状态。

这些信息可以分布在边角或斜栏内部，避免遮挡地球操作。

## 5. 飞线 hover 小窗口

飞线 hover 小窗口用于快速解释当前攻击路径。

触发方式：

- 鼠标靠近来源点。
- 鼠标靠近飞线。
- 鼠标准星对准可交互节点。

展示内容：

- 来源地到成都，例如 `Frankfurt -> Chengdu`。
- 风险等级。
- 攻击类型。
- Cloudflare action。
- 请求路径摘要。
- 时间。

设计要求：

- 不使用厚重背景卡片。
- 不使用大面积彩色面板。
- 使用森雨风格的角标、扫描、细线、短字信息。
- 小窗口必须做边缘适配，不能被导航遮挡。

## 6. 态势页侧栏信息

### 左侧控制

- 时间窗口。
- 数据模式。
- 风险等级。
- 攻击类型。
- 来源地区。
- 图层开关。
- 2D / 3D 视图切换。

### 右侧分析

- Top 攻击类型。
- Top 来源地区。
- 地区攻击习惯。
- 工具指纹。
- 规则命中趋势。
- 新异常聚合。
- 当前数据可信度。

## 7. 响应式要求

主要适配：

- 1920×1080。
- 2560×1440。
- 3840×2160。

同时保证移动宽度不横向溢出。

桌面端保持主视觉和斜向导航；移动端可以降级为纵向结构，但必须保留项目视觉特征。

## 8. 视觉限制

禁止回到以下问题：

- 大面积绿色仪表盘。
- 卡片堆叠。
- 组件随意摆放。
- 首页放太多运营面板。
- 每个页面导航逻辑不同。
- 仿真页被无关 HUD 遮挡。
- 内容越界或被导航盖住。

所有功能页面必须保持统一位置逻辑：顶部或侧边提供导航，主体区域承载唯一核心任务。
