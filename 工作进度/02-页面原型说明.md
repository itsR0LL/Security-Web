# 博客安全态势监控平台页面原型说明 v0.1

## 1. 页面原则

首页采用“一屏总览 + 下钻列表”的产品结构。

3D 地球是首页强视觉模块，但不替代安全分析主流程。用户必须能从总览卡片、图表、地球来源点、最近事件等入口下钻到事件列表和事件详情。

页面风格后续贴近现有博客管理后台：Next.js、React、Tailwind、lucide-react、framer-motion，并保持私有后台工具的清晰、克制和可扫描。

## 2. 页面清单

一期页面：

1. `/security`
   - 安全态势首页
   - 一屏总览
   - 3D 地球
   - 流量分析
   - 数据同步状态
   - 最近事件入口

2. `/security/events`
   - 事件列表
   - 强筛选
   - 时间倒序
   - 事件详情抽屉

3. `/security/events/[id]`
   - 事件详情页或详情视图
   - 原始数据
   - 规则命中
   - Cloudflare 处理结果
   - AI 分析占位

4. `/security/settings`
   - Cloudflare 配置
   - Token 权限检测
   - 刷新周期
   - 高风险提醒阈值
   - SQLite 保留策略
   - 样例模式状态

## 3. 首页结构

首页首屏包含：

- 顶部状态栏：平台名称、监控域名、时间范围、手动刷新、样例模式提示、设置入口。
- 核心总览区：访问态势、攻击态势、异常来源、数据同步状态。
- 3D 地球主视觉区：来源点与飞线统一指向成都。
- 辅助分析区：流量分析、最近事件摘要。
- 下钻入口：点击任一指标、来源点、趋势点或事件进入筛选后的列表或详情。

首页信息优先级：

1. 当前是否异常：高风险事件、攻击数量、异常来源数、最近严重事件。
2. 趋势是否变化：访问量、攻击量、拦截量、挑战量、错误率。
3. 来源在哪里：3D 地球、异常来源榜、国家/地区分布。
4. 发生了什么：最近事件摘要、事件类型分布。
5. 数据是否可信：同步时间、同步状态、权限状态、样例数据提示。

## 4. 首页组件

核心组件：

- KPI 卡片：访问量、攻击量、拦截数、异常来源数、高风险事件数。
- 趋势图：访问态势、攻击态势、流量分析。
- 流量分析：请求量、带宽、缓存命中、状态码、热门路径、来源国家/地区。
- 异常来源榜：国家/地区、IP、请求数、风险等级。
- 最近事件列表：时间、来源、路径、类型、Cloudflare action、风险等级。
- 数据同步状态卡：最近同步时间、刷新周期、接口状态、权限检测结果、本地事件数量。
- 3D 地球：来源点、飞线、目的地成都、风险颜色编码。

3D 地球交互：

- 支持旋转、缩放、悬停提示。
- 来源点按风险等级或事件类型着色。
- 飞线统一指向成都。
- 点击来源点进入该来源筛选后的事件列表。
- 点击飞线或来源点展示来源 IP、地区、风险等级、请求量、定位精度。
- WebGL 不可用时降级为二维来源分布图。

## 5. 事件列表

列表默认规则：

- 默认按时间倒序。
- 强筛选始终可见或可固定。
- 筛选变更后，列表、分页和统计摘要同步更新。
- 支持清空筛选，但保留默认时间范围。

筛选项：

- 时间范围
- 风险等级
- 事件类型
- 来源 IP
- 来源国家/地区
- 请求路径关键词
- User-Agent 关键词
- 请求方法
- 状态码
- Cloudflare action

列表字段：

- 时间
- 来源
- 国家/地区
- 请求方法
- 路径
- 状态码
- 事件类型
- Cloudflare action
- 风险等级
- 定位精度

详情抽屉：

- 请求基础信息
- 来源定位信息
- 风险判断原因
- 命中规则
- Cloudflare 处理结果
- 原始事件 JSON
- AI 分析占位

## 6. 设置页

设置页包含：

- Cloudflare Zone ID
- 监控域名：`r0l1dehome.asia`
- Cloudflare API Token
- Token 权限检测结果
- 自动刷新周期，默认 6 小时
- 高风险提醒阈值
- SQLite 原始事件保留周期，默认 90 天
- 聚合统计长期保留说明
- 样例模式状态说明

交互规则：

- 保存 Token 后立即执行权限检测。
- 权限检测至少检查 Zone、Analytics、Security Events 是否可读。
- 权限不足时明确提示缺失项。
- 无 Token 时自动启用样例数据。
- Token 无效或权限不足时不自动切换为样例数据。
- 同步失败时保留上一次成功数据，并显示数据可能过期。

## 7. 状态设计

数据状态：

- 加载中：局部 skeleton 或 loading。
- 正常：展示真实 Cloudflare 数据。
- 无 Token：自动启用样例数据，并明确标记。
- Token 无效：显示配置错误，不伪装为样例数据。
- 权限不足：显示可连接但数据不完整。
- 同步失败：保留上次成功数据，展示失败原因。
- 无数据：展示空状态和配置/筛选建议。

风险状态：

- `info`：信息类。
- `low`：低风险。
- `medium`：需要关注。
- `high`：高风险。
- `critical`：严重风险。

所有风险状态必须同时使用颜色、文字和图标表达，不能只依赖颜色。

## 8. 响应式

桌面端：

- 首页首屏完整展示 KPI、3D 地球、关键图表和最近事件。
- 事件列表使用表格 + 右侧详情抽屉。

平板端：

- KPI 卡片换行。
- 3D 地球与趋势图上下排列。
- 事件详情使用底部或右侧抽屉。

移动端：

- 首页改为纵向信息流。
- 3D 地球可降级为简化地图或来源列表。
- 筛选区折叠为筛选面板。
- 事件列表改为卡片式展示。

## 9. 可访问性

- 图表和 3D 地球必须提供文本摘要。
- 关键操作支持键盘访问。
- 表单字段有明确 label、错误提示和帮助说明。
- 同步失败、Token 无效、权限不足等状态必须有明确提示。
- 动效尊重系统“减少动态效果”设置。

## 10. 验收要点

- 首页必须实现一屏总览，不应变成纯报表列表页。
- 首页必须存在到事件列表和事件详情的下钻路径。
- 3D 地球必须展示来源点，并通过飞线指向成都。
- 事件列表必须按时间倒序展示。
- 事件列表必须具备强筛选能力。
- 设置页必须覆盖 Cloudflare 配置、权限检测、刷新周期、风险阈值、数据保留、样例模式。
- 配置 Token 后，即使同步失败或权限不足，也不能自动切换成样例数据。
- 数据同步状态必须让用户判断当前数据是否可信、是否最新。
