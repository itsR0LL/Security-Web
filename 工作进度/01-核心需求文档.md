# Security Studio 核心需求文档 v2.0

更新时间：2026-06-03

## 1. 项目定位

Security Studio 是面向个人博客 `r0l1dehome.asia` 的私有安全态势与攻击行为分析平台。

它不是公开大屏，也不是企业 SIEM。当前目标是：

- 监控博客访问与 Cloudflare 安全事件。
- 将访问日志、边缘安全事件、规则命中结果统一展示。
- 用规则研判识别攻击类型、风险等级、工具指纹和行为特征。
- 将一段时间内的攻击行为聚合成可分析的行为组。
- 为后续 AI 规则建议、人工审核和规则版本迭代预留结构。

当前按 `E:\Project\Security Studio` 作为本地工作目录执行。

## 2. 当前已完成范围

### 页面

- `/security`：保留森雨风格首页、斜向导航、准星、粒子地球主视觉。
- `/security/situation`：保留 3D / 2D 态势仿真，并增强右侧安全态势侧栏。
- `/security/events`：事件筛选、事件列表、事件详情、规则命中、告警文本。
- `/security/events/[id]`：单事件详情，读取 `GET /api/events/{event_id}`。
- `/security/settings`：Cloudflare 配置、Token 状态、同步策略、风险阈值。
- `/security/map`：2D 请求分布补充页面。
- `/security/analysis`：二期新增攻击聚合分析页面。

### 数据链路

- Cloudflare GraphQL 基础数据链路。
- Worker + D1 访问日志链路。
- FastAPI + SQLite 本地存储。
- 样例数据 fallback。
- 真实 Worker 日志同步到本地 SQLite。

### 研判能力

- 可版本化规则结构。
- RuleMatcher 第一版。
- 事件攻击分类字段。
- 规则命中证据字段。
- AttackAggregator 第一版。
- 非 LLM 的规则建议草案。

## 3. 当前核心功能

### 访问态势

- 请求量、来源国家/地区、来源 IP、路径、状态码、带宽与缓存相关指标。
- Worker 日志中的普通访问进入访问态势和来源统计。

### 攻击态势

- Cloudflare action、路径探测、SQL 注入探测、XSS 探测、可疑 UA 等规则命中。
- 攻击事件进入 `/security/events` 与 `/security/analysis`。
- 正常访问不进入攻击行为组。

### 攻击聚合

二期聚合维度包括：

- 时间窗口。
- 国家/地区/城市。
- IP、ASN。
- 路径、方法、状态码。
- User-Agent。
- Cloudflare action。
- 规则命中。
- 攻击类型、风险等级。

输出为攻击行为组，包含事件数、时间范围、主要来源、主要路径、主要工具指纹、主要规则、风险等级、置信度和代表证据。

## 4. 当前明确不做

- 不接真实大模型。
- 不自动启用规则。
- 不自动封禁 IP。
- 不直接修改 Cloudflare WAF 配置。
- 不接主机日志、系统日志和攻击链还原。
- 不重做首页主视觉、地球粒子、飞线动效、开屏动画。

## 5. 下一阶段方向

第三期重点是“规则建议治理”：

```text
/security/analysis 聚合结果
-> 规则建议草案
-> 人工审核状态
-> shadow mode 回放验证
-> 规则版本记录
-> 人工确认后启用
```

后续再进入主机日志、攻击链还原、攻击基础设施画像等更深层能力。
