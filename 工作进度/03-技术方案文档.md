# 博客安全态势监控平台技术方案文档 v0.1

## 1. 技术目标

在 `E:\Project\Secrity Studio` 内先作为独立项目开发一个私有安全态势监控平台，稳定后再考虑接入 `E:\Project\my-blog-manager`。

一期目标：

- 接入 Cloudflare 免费版只读数据。
- 监控域名 `r0l1dehome.asia`。
- 6 小时同步一次数据。
- 使用 SQLite 保留 90 天原始事件，长期保留聚合统计。
- 提供安全态势首页、事件列表、事件详情、设置页。
- 使用 Three.js 实现 3D 地球来源点与飞线到成都。
- 无 Cloudflare Token 时自动启用样例数据。
- 同步失败时保留上一次成功数据。

## 2. 总体架构

建议沿用现有博客管理后台的技术方向：

- 前端：Next.js + React + TypeScript + Tailwind CSS
- 图标与动效：lucide-react + framer-motion
- 3D 可视化：Three.js 或基于 Three.js 的 React 封装
- 后端：FastAPI
- 存储：SQLite
- 数据源：Cloudflare GraphQL Analytics API / Security Events，只读
- 鉴权边界：本地管理 token，后续可复用 `my-blog-manager` 的 `X-Manager-Token` 模式

逻辑边界：

- 前端只负责展示、筛选、触发同步、配置输入。
- 后端负责 Cloudflare 调用、Token 权限检测、规则分析、地理定位补全、聚合统计、SQLite 写入。
- Token、Zone ID 等敏感配置只在后端本地存储，不进入前端源码，不写入公开博客仓库。

## 3. 模块划分

### 3.1 前端模块

- 安全态势总览
- 3D 地球可视化
- 流量分析
- 异常来源分析
- 最近事件摘要
- 事件列表与筛选
- 事件详情抽屉或详情页
- 数据同步状态
- Cloudflare 设置与权限检测
- 样例数据状态提示

### 3.2 后端模块

- `CloudflareClient`
  - 封装 GraphQL 请求、分页、限流、错误处理。

- `SyncService`
  - 每 6 小时同步。
  - 支持手动触发同步。
  - 同步失败时保留旧数据。

- `EventNormalizer`
  - 将 Cloudflare 数据统一为内部事件模型。

- `GeoResolver`
  - 优先使用 Cloudflare 地理字段。
  - 缺失时用本地 IP 地理库补全。
  - 再缺失时按地区/国家生成稳定伪随机点。

- `RuleEngine`
  - 使用内置默认规则计算风险等级和事件类型。
  - 一期不开放页面化规则编辑，只预留数据结构。

- `AggregationService`
  - 生成小时/天级聚合。
  - 长期保留聚合统计。

- `SampleDataService`
  - 无 Token 时自动启用样例数据。
  - Token 无效或权限不足时不伪装为样例数据。

- `ApiLayer`
  - 向前端提供总览、事件、趋势、地图、同步状态、设置接口。

## 4. 数据流

1. 用户在 `/security/settings` 配置 Cloudflare Zone ID 和 API Token。
2. 后端保存配置，并立即执行 Token 权限检测。
3. 权限检测通过后，系统按 6 小时周期同步 Cloudflare 数据。
4. 同步服务拉取访问统计、安全事件和相关聚合数据。
5. 后端将 Cloudflare 数据标准化为内部事件模型。
6. 规则引擎为事件打标签并计算风险等级。
7. 地理定位模块为事件生成 3D 地球来源点。
8. 原始事件写入 SQLite，聚合服务刷新统计表。
9. 前端读取总览、趋势、地图、事件列表和同步状态。
10. 同步失败时，前端继续展示上一次成功数据，并提示数据可能过期。

## 6. API 设计

### 6.1 基础接口

- `GET /api/health`
  - 服务健康检查。

- `GET /api/overview`
  - 返回访问量、威胁量、拦截量、风险等级分布、最近同步状态。

- `GET /api/sync/status`
  - 返回最近同步任务、最后成功同步时间、是否使用旧数据、API 错误、本地事件数量。

- `POST /api/sync/run`
  - 手动触发同步。

### 6.2 事件接口

- `GET /api/events`
  - 参数：`from`、`to`、`ip`、`country`、`region`、`risk_level`、`event_type`、`action`、`path`、`status_code`、`limit`、`offset`
  - 默认按 `occurred_at desc` 返回。

- `GET /api/events/{id}`
  - 返回事件详情、原始 JSON、规则命中、定位精度、AI 占位信息。

### 6.3 聚合接口

- `GET /api/aggregates/trends`
  - 返回访问、攻击、状态码、流量趋势。

- `GET /api/aggregates/sources`
  - 返回来源国家/地区/IP 排名。

- `GET /api/aggregates/map`
  - 返回 3D 地球所需来源点、飞线、风险等级、事件数量。
  - 目的地固定为成都。

### 6.4 设置接口

- `GET /api/settings`
  - 返回当前设置，敏感 Token 只返回是否已配置。

- `POST /api/settings/cloudflare`
  - 保存 Zone ID、监控域名、API Token。

- `POST /api/token/check`
  - 检测 Cloudflare Token 权限。

- `POST /api/settings/risk-threshold`
  - 设置高风险提醒阈值。

### 6.5 规则接口

一期不实现页面化规则编辑。后端可预留：

- `GET /api/rules`

二期再考虑：

- `POST /api/rules`
- `PATCH /api/rules/{id}`

## 7. 同步任务

周期：

- 默认每 6 小时执行一次。
- 支持手动同步。

同步窗口：

- 首次同步优先拉取最近 24 小时。
- 后续从最后成功同步时间继续。
- 受 Cloudflare 免费版限制，Security Events 明细必须及时落地本地 SQLite。

失败策略：

- 不清空旧数据。
- 写入 `sync_runs.status = failed`。
- 前端展示“数据截至上次成功同步”。
- Token 缺失时启用样例数据。
- Token 无效或权限不足时展示配置/权限错误，不自动伪装成样例数据。

保留策略：

- 原始事件保留 90 天。
- 90 天前原始事件按天汇总后清理。
- 聚合统计长期保留。

## 8. 规则引擎

一期内置默认规则：

- 敏感路径探测：`.env`、`wp-login.php`、`phpmyadmin`、`admin` 等。
- 疑似 SQL 注入关键词。
- 疑似 XSS 关键词。
- 高频 IP。
- 高频 404。
- 可疑 User-Agent。
- Cloudflare `block`、`challenge`、`managed_challenge` 等 action。

输出：

- `event_type`
- `risk_level`
- `confidence`
- `summary`
- 高风险待发送消息文本

一期设置页只允许调整高风险提醒阈值。自定义规则编辑器作为二期能力。

## 9. 地理定位与 3D 地球

目的地：

- 成都。

来源点优先级：

1. Cloudflare 返回城市、经纬度。
2. 使用来源 IP 通过本地地理库补全。
3. 使用省/州/地区范围内稳定伪随机点。
4. 使用国家范围内稳定伪随机点。
5. 只有聚合国家统计时，展示国家级热力点或国家级飞线。

稳定随机要求：

- 伪随机点基于事件 ID、IP、国家/地区等生成。
- 同一来源每次刷新位置应稳定。
- UI 标记定位精度：城市、地区、国家、估算。

降级：

- WebGL 不可用时降级为二维来源分布图。

## 10. 样例数据模式

触发条件：

- 未配置 Cloudflare Token。

不触发条件：

- Token 无效。
- 权限不足。
- 同步失败。

要求：

- 首页和设置页明确标记“样例数据”。
- 样例数据覆盖 KPI、趋势、事件列表、3D 地球、流量分析。
- 配置 Token 后自动退出样例模式。

## 11. 风险与约束

- Cloudflare 免费版 Security Events 历史窗口短，必须及时本地沉淀。
- Security Events 可能采样，不能承诺完整请求级取证。
- 普通访问统计更适合聚合展示，不应宣传为完整原始访问日志。
- IP 地理位置存在误差，3D 地球用于态势展示，不作为取证依据。
- SQLite 适合本地轻量工具，但需要定期清理原始事件。
- Token 权限配置错误会导致同步失败或数据缺失。
- 样例数据必须明显标识，避免与真实监控数据混淆。

## 12. 验证策略

一期验证重点：

- 无 Token 时样例数据自动展示。
- 配置 Token 后执行权限检测。
- 同步失败时保留旧数据。
- 事件列表默认时间倒序。
- 筛选条件组合生效。
- 3D 地球有来源点和飞线到成都。
- SQLite 原始事件保留和聚合统计写入正常。
- Token 不进入前端源码或公开博客仓库。
