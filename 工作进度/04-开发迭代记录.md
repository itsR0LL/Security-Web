## 4. 决策组整理后的下一阶段方向

下一阶段核心不是继续改视觉，而是建立攻击研判与规则迭代能力。

统一方向：

```text
日志事件
-> 标准化事件
-> 规则命中
-> 特征提取
-> 攻击聚合
-> AI 辅助归类与建议
-> 人工审核
-> 规则版本更新
-> 可视化反馈
```

当前优先级：

1. 规则 JSON。
2. 事件特征字段。
3. 攻击聚合。
4. 分析页面。
5. AI 规则建议草案。

## 5. 下一轮开发任务

### 任务 1：规则 JSON 类型固定

目标：

- 建立规则类型定义。
- 明确规则字段。
- 支持版本号、启用状态、shadow mode、分类、证据、动作、审核状态。

产出：

- 前后端统一规则结构。
- 样例规则文件或 SQLite 初始化数据。

### 任务 2：事件模型扩展

目标：

- 给事件加入攻击分类、规则命中、工具指纹、聚合 ID、置信度等字段。
- 修改样例数据，保证页面可展示下一阶段内容。

产出：

- 扩展后的事件结构。
- 事件页展示规则命中与攻击分类。

### 任务 3：RuleMatcher 第一版

目标：

- 使用规则 JSON 判断事件。
- 输出规则命中、风险等级、攻击类型、证据。
- 支持 shadow mode。

产出：

- 后端规则匹配服务。
- 事件同步或样例生成时可写入规则命中。

### 任务 4：AttackAggregator 第一版

目标：

- 按时间、来源、路径、UA、Cloudflare action、规则命中进行聚合。
- 输出攻击行为组。

产出：

- 聚合接口。
- 攻击类型趋势。
- 来源地区攻击习惯。
- 工具指纹摘要。

### 任务 5：新增 `/security/analysis`

目标：

- 展示攻击类型矩阵。
- 展示来源地区习惯。
- 展示规则命中趋势。
- 展示未知攻击聚合。
- 展示 AI 规则建议草案。

产出：

- 可用的分析页 MVP。

### 任务 6：增强 `/security/situation` 侧栏

目标：

- 不改地球主视觉。
- 在侧栏展示攻击分类、来源习惯、规则趋势、数据可信度。
- 让点击来源或攻击类型可以跳转到事件页或分析页。

产出：

- 态势页从纯视觉转为安全态势入口。

## 7. 当前风险

- Cloudflare 免费版数据存在采样和历史窗口限制。
- AI 输出存在误判风险，必须人工审核。
- 攻击来源只能做基础设施和行为画像，不能直接断言真实身份。
- 页面风格容易压过功能，需要控制信息密度。
- 后续服务器日志涉及隐私、权限和安全边界，需要独立设计。
